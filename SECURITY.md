# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Proscan, we ask that you report it responsibly. Do not open a public GitHub issue for security vulnerabilities.

**Email:** [security@proscan.one](mailto:security@proscan.one)

When reporting, include as much detail as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce (detailed enough for us to verify)
- The version of Proscan you're running
- Any relevant logs, screenshots, or proof-of-concept code

If you have a PGP key and want to encrypt your report, contact us first at the address above and we'll share our public key.

## What to Expect

- **Acknowledgment** — we'll confirm receipt within 48 hours
- **Assessment** — we'll investigate and determine the severity and scope
- **Updates** — we'll keep you informed of progress toward a fix
- **Fix and disclosure** — once resolved, we'll release a patch and (with your permission) credit you in the advisory

We aim to resolve confirmed vulnerabilities as quickly as possible. Timelines depend on the complexity of the issue, but we treat security reports as high priority.

## Supported Versions

Security fixes are provided for the current release. We recommend keeping Proscan updated to the latest version.

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Previous releases | Case-by-case |

## Scope

This policy covers the Proscan application — the backend, frontend, launcher, and any associated tooling distributed as part of the product.

It does not cover:

- Third-party dependencies (report those to the respective maintainers)
- The proscan.one website infrastructure (contact us directly for those)
- Social engineering or physical attacks

## Safe Harbor

We consider security research conducted under this policy to be authorized. We will not pursue legal action against researchers who:

- Report vulnerabilities through the process described above
- Avoid accessing or modifying other users' data
- Do not disclose the vulnerability publicly before it's resolved
- Make a good-faith effort to avoid disruption to Proscan users
